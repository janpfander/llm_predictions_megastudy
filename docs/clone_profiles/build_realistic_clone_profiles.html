1 Overview

This document builds the profile bank used to prompt the LLM clones for our climate-trust megastudy. The main idea here is to use the demographics and attitudes of real human personas. This avoids creating fictional personas based on quotas (e.g., the Black, college-educated, evangelical Republican rancher in rural Maine). A major advantage is that, for the real personas, we do not only have demographic data, but also potentially relevant data on attitudes and beliefs (e.g., trust in scientists in general). The structure of this script is the following:

  1. Loads the most recent (2024) General Social Survey (GSS) cross-section.
  2. Selects demographic and opinion variables that match (or most closely correspond to) the items in our human survey.
  3. Drops respondents with missing values on any selected variable (no imputation).
  4. Draws N = 500 unique profiles by weighted sampling without replacement, with selection probability proportional to each respondent’s post-stratification weight.
  5. Generates two parallel sets of prompt profiles — a basic one (Hewitt-style minimal demographics) and a detailed one (full demographic + opinion profile) — so we can compare prompt richness as a robustness analysis.

Each of the 500 profiles will be queried in every experimental condition, in independent API sessions. This way, we have no sampling noise, and the different conditions represent actual counterfactuals for the 500 different profiles.

2 Design rationale

We use the most recent wave of the GSS (2024) rather than Census data as the source for the profiles because GSS contains both the demographic variables we need and the most directly relevant opinion variable for our outcome (confsci, confidence in the scientific community). ACS PUMS would buy us finer demographic granularity but no attitudes.

We will use Weighted sampling without replacement. Draw N=500 unique rows, probability proportional to weight. Every donor has a nonzero chance of being in the bank, no donor appears twice, and the realized marginal distributions converge to the weighted GSS marginals.

Per-condition N is therefore 500 across all 21 conditions, yielding 500 × 21 = 10,500 clone responses per outcome. This corresponds to roughly half of the human sample size per condition.

We generate two parallel prompt sets:

  • Basic profile: age, sex, race, education, party identification.
  • Detailed profile: all basic profile variables, as well as income, social class, urban/rural, religion, religiosity, political ideology, and confidence in the scientific community.

Whether richer profiles improve clone accuracy is empirically open. Argyle et al. found rich profiles helped; Wang et al. and Bisbee et al. found that more identity information can produce flattened, stereotyped responses.

6 Step 3. Drop missing data

Respondents must be non-missing on the post-stratification weight and on the core demographic variables, but NA is allowed on opinion / intensity variables and is handled later in the prompt builder.

Weight NAs are structural, not random. wtssps is missing for ~17% of GSS 2024 respondents because they could not be placed in a post-stratification cell (typically due to refusing one of the post-strat variables themselves). We cannot weight these respondents, so they have to come out.

6.1 Step 3a. Recode GSS missing-value labels to proper NA

In GSS labelled data, missingness appears as named factor levels rather than real NAs. We recode the four common labels ("don't know", "no answer", "skipped on web", "iap") before applying the missing-data rule.

gss_missing_labels <- c("no answer", "skipped on web")

gss_sel <- gss_sel |>
  mutate(across(where(is.factor), ~ {
    chr <- as.character(.x)
    chr[tolower(chr) %in% gss_missing_labels] <- NA
    factor(chr, levels = setdiff(levels(.x), gss_missing_labels))
  }))

6.2 Step 3b. Tiered missing-data rule

# Required: respondents missing any of these are dropped.

7 Step 4. Weighted sampling without replacement

We draw 500 unique GSS respondents, with selection probability proportional to their wtssps post-stratification weight.

target_n <- 500

profiles <- gss_complete |>
  slice_sample(n = target_n, weight_by = wtssps, replace = FALSE) |>
  mutate(profile_id = sprintf("p%04d", row_number())) |>
  select(profile_id, everything())

glue("Profile bank built: N = {nrow(profiles)} unique clones from ",
     "{nrow(gss_complete)} eligible GSS respondents.")

Because sampling is random, the script is reproducible only with the seed set in the setup (set.seed(2026)). Re-running with a different seed will produce a different 500-profile bank.

8 Step 5. Validate the profile bank against weighted GSS

A sanity check: the marginal distribution of each variable in profiles should approximate the weighted marginal distribution in the GSS source. With N = 500 there will be sampling variation, so deviations of a few percentage points on small categories are expected. We compute side-by-side summaries.

validate_marginal <- function(var) {
  src <- gss_complete |>
    count(.data[[var]], wt = wtssps) |>
    mutate(prop_gss = n / sum(n)) |>
    select(-n)
  prof <- profiles |>
    count(.data[[var]]) |>
    mutate(prop_profiles = n / sum(n)) |>
    select(-n)
  full_join(src, prof, by = var) |>
    mutate(diff = prop_profiles - prop_gss) |>
    mutate(across(c(prop_gss, prop_profiles, diff), ~ round(.x, 4)))
}

vars_to_check <- selected_vars

walk(vars_to_check, \(v) {
  cat("\n----", v, "----\n")
  print(validate_marginal(v))
})

With N = 500 drawn from a donor pool of ~2,000+, the largest absolute deviation on a multi-category variable will typically be a few percentage points. Systematically large deviations (e.g., > 5 pp on a major category) indicate either a coding error or that the seed produced an unrepresentative draw.

9 Step 6. Construct prompt-profile strings

We build two parallel profile strings per clone: a basic one and a detailed one. Each is a self-contained natural-language description of the simulated respondent. The downstream simulation script will concatenate this string with (a) a role/framing message, (b) the experimental stimulus, and (c) the outcome question.

The detailed version retains GSS labels verbatim. We write small helpers that translate factor levels into natural prose, but we do not recode GSS values into the human-survey response categories — that mapping is a separate step.

library(purrr)   # for compact()

lbl <- function(x) {
  v <- if (is.factor(x)) as.character(x) else as.character(x)
  if (length(v) == 0 || is.na(v)) NA_character_ else v
}

# Returns NULL when value is NA, so compact() can drop the line
opt_line <- function(label, value, transform = tolower) {
  v <- lbl(value)
  if (is.na(v)) return(NULL)
  glue("- {label}: {transform(v)}.")
}

format_relig_line <- function(relig, reliten, attend, reborn) {
  r <- lbl(relig)
  if (is.na(r)) return(NULL)
  if (tolower(r) == "none") return("- Religion: not religious.")
  parts <- glue("- Religion: {tolower(r)}.")
  rl <- lbl(reliten); if (!is.na(rl)) parts <- c(parts, glue("- Strength of religious affiliation: {tolower(rl)}."))
  at <- lbl(attend);  if (!is.na(at)) parts <- c(parts, glue("- Religious service attendance: {tolower(at)}."))
  rb <- lbl(reborn);  if (!is.na(rb) && tolower(rb) == "yes") {
    parts <- c(parts, "- Self-identifies as a born-again or evangelical Christian.")
  }
  paste(parts, collapse = "\n")
}

build_basic_profile <- function(row) {
  glue(
    "You are simulating the response of a U.S. resident with the following ",
    "characteristics: ",
    "age {row$age}; ",
    "sex {tolower(lbl(row$sex))}; ",
    "race {tolower(lbl(row$race))}; ",
    "highest educational degree: {tolower(lbl(row$degree))}; ",
    "party identification: {tolower(lbl(row$partyid))}."
  )
}

build_detailed_profile <- function(row) {
  lines <- list(
    "You are simulating the response of a U.S. resident with the following characteristics.",
    glue("- Age: {row$age} years old."),
    glue("- Sex: {tolower(lbl(row$sex))}."),
    glue("- Race: {tolower(lbl(row$race))}."),
    opt_line("Hispanic origin",                            row$hispanic),
    glue("- Highest educational degree: {tolower(lbl(row$degree))}."),
    glue("- Household income (GSS bracket): {tolower(lbl(row$income16))}."),
    opt_line("Household size",                             row$hompop, transform = identity),
    glue("- Self-described social class: {tolower(lbl(row$class))}."),
    glue("- Census region: {tolower(lbl(row$region))}."),
    glue("- Party identification: {tolower(lbl(row$partyid))}."),
    glue("- Political views (liberal-conservative): {tolower(lbl(row$polviews))}."),
    format_relig_line(row$relig, row$reliten, row$attend, row$reborn),
    opt_line("Confidence in the scientific community",     row$consci)
  )
  paste(compact(lines), collapse = "\n")
}
profiles <- profiles |>
  rowwise() |>
  mutate(
    profile_basic    = build_basic_profile(pick(everything())),
    profile_detailed = build_detailed_profile(pick(everything()))
  ) |>
  ungroup()

write_csv(profiles, path(out_dir, "profiles.csv"))

  paste(parts, collapse = "\n")
}

build_basic_profile <- function(row) {
  glue(
    "You are simulating the response of a U.S. resident with the following ",
    "characteristics: ",
    "age {row$age}; ",
    "sex {tolower(lbl(row$sex))}; ",
    "race {tolower(lbl(row$race))}; ",
    "highest educational degree: {tolower(lbl(row$degree))}; ",
    "party identification: {tolower(lbl(row$partyid))}."
  )
}

build_detailed_profile <- function(row) {
  lines <- list(
    "You are simulating the response of a U.S. resident with the following characteristics.",
    glue("- Age: {row$age} years old."),
    glue("- Sex: {tolower(lbl(row$sex))}."),
    glue("- Race: {tolower(lbl(row$race))}."),
    opt_line("Hispanic origin",                            row$hispanic),
    glue("- Highest educational degree: {tolower(lbl(row$degree))}."),
    glue("- Household income (GSS bracket): {tolower(lbl(row$income16))}."),
    opt_line("Household size",                             row$hompop, transform = identity),
    glue("- Self-described social class: {tolower(lbl(row$class))}."),
    glue("- Census region: {tolower(lbl(row$region))}."),
    glue("- Party identification: {tolower(lbl(row$partyid))}."),
    glue("- Political views (liberal-conservative): {tolower(lbl(row$polviews))}."),
    format_relig_line(row$relig, row$reliten, row$attend, row$reborn),
    opt_line("Confidence in the scientific community",     row$consci)
  )
  paste(compact(lines), collapse = "\n")
}

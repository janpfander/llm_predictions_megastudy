Predicting the effects of behavioral interventions on humans with LLMs

  • Show All Code

Published

May 4, 2026

Abstract

We test the potential of predicting human behavior in survey experiments with behavioral clones—large language model (LLM) generated artificial agents mimicking human participants. To evaluate this potential, we predict the outcomes of a large-scale field experiment (N = ~22,000) to be conducted in the United States. The experiment will test 20 text-based interventions designed to increase trust in climate scientists, assessing a broad range of secondary, outcomes including self-reported attitudes (e.g., support for funding of climate science) and behavioral measures (e.g., donations to a scientific organization). Before the human study is fielded, we will simulate it with behavioral clones using demographically representative U.S. participant profiles. We will lock the simulated dataset and analysis code. Once human data are available, we will compare clone-based predictions to human results using a set of preregistered metrics spanning (i) inferential agreement and estimate convergence on treatment effects, (ii) outcome distributions, and (iii) subgroup heterogeneity. Our study will provide a large-scale, credible benchmark test of LLMs’ potential—and limitations—as tools for simulating human subjects in social science research. All analyses in this preregistration are demonstrated on simulated placeholder data with identical structure to the real datasets; no human outcome data have been observed at the time of registration.

Research Questions

The main outcome of the megastudy is trust in climate scientists. However, there are several secondary and tertiary outcomes. For each of these outcomes we have the following research questions:

RQ1 — ATE recovery. How well do LLM clones predict the average treatment effects (ATEs) across 20 interventions?

RQ2 — Distribution shape recovery. How well do LLM clones mirror human response distributions (within experimental conditions)?

RQ3 — Subgroup heterogeneity recovery. How well do LLM clones recover differences in intervention effects across different demographic groups?

RQ4 — Individual-level demographic calibration. Are LLM clone responses calibrated to the demographic profile of each simulated participant? We examine (i) whether clones reproduce the correct baseline response level for each demographic group, (ii) whether demographic variables predict clone responses with the same magnitude as in humans (stereotyping test), and (iii) whether within-subgroup response distributions match between humans and clones.

Secondary — Stacked-model discrepancy test. As a complement to the TOST-based equivalence tests in RQ1, we pool human and clone data and estimate a condition × source interaction, providing a direct inferential test of whether each clone ATE significantly departs from its human counterpart.

Comparison Functions

Two helper functions operationalize the comparison metrics. They are applied symmetrically to human and LLM clone model outputs throughout the document.

Comparison metrics

compare_estimates() is used throughout the document — for ATEs (RQ1), moderator interaction estimates (RQ3), and secondary analyses. It takes two tidy model result tibbles and a join_by argument specifying the key columns, and computes six metrics:

  • Spearman ρ — rank correlation of point estimates; primary metric for RQ1
  • Pearson r — linear correlation of point estimates
  • RMSE — root mean squared error; captures magnitude accuracy
  • Directional agreement — % of estimates with the same sign
  • Inferential agreement — % of estimates with the same BH-adjusted conclusion (significantly positive, significantly negative, or non-significant at α = .05)
  • TOST (%) — % of individual estimate pairs that pass an equivalence test with preregistered bound Δ_k = 0.5 × |estimate_human_k|; equivalence is declared (two one-sided tests, α = .05) when the clone estimate falls within ±50% of the human estimate. Note: when human estimates are near zero, Δ_k shrinks accordingly, making equivalence harder to achieve.
Code 
compare_estimates <- function(h_result, l_result,
                              join_by = "condition",
                              tost_delta_pct = 0.5) {

  # Standardize columns; SE derived from std.error or from 95% CI width if absent
  prep <- function(df, suffix) {
    df |>
      select(all_of(join_by), estimate) |>
      mutate(
        se    = if ("std.error" %in% names(df)) df$std.error
                else if (all(c("conf.low", "conf.high") %in% names(df)))
                  (df$conf.high - df$conf.low) / (2 * 1.96)
                else NA_real_,
        p_adj = if ("p.value_adjusted" %in% names(df)) df$p.value_adjusted
                else NA_real_
      ) |>
      rename_with(~ paste0(.x, "_", suffix), c(estimate, se, p_adj))
  }

  joined <- inner_join(prep(h_result, "h"), prep(l_result, "l"), by = join_by)

  # Three-level inferential category based on BH-adjusted p and direction
  infer_cat <- function(est, p_adj) {
    case_when(
      p_adj < 0.05 & est > 0 ~ "sig_positive",
      p_adj < 0.05 & est < 0 ~ "sig_negative",
      TRUE                   ~ "not_significant"
    )
  }

  # TOST: equivalence declared when |ATE_h - ATE_l| < delta = tost_delta_pct × |ATE_h|
  # p_tost is the maximum of the two one-sided p-values; equivalent if p_tost < 0.05
  joined |>
    mutate(
      delta      = tost_delta_pct * abs(estimate_h),
      diff       = estimate_h - estimate_l,
      se_diff    = sqrt(se_h^2 + se_l^2),
      p_lower    = pnorm((diff + delta) / se_diff, lower.tail = FALSE),
      p_upper    = pnorm((diff - delta) / se_diff),
      p_tost     = pmax(p_lower, p_upper),
      equivalent = p_tost < 0.05,
      same_infer = infer_cat(estimate_h, p_adj_h) == infer_cat(estimate_l, p_adj_l)
    ) |>
    summarise(
      spearman_rho    = cor(estimate_h, estimate_l, method = "spearman",
                            use = "pairwise.complete.obs"),
      pearson_r       = cor(estimate_h, estimate_l, use = "pairwise.complete.obs"),
      rmse            = sqrt(mean((estimate_h - estimate_l)^2, na.rm = TRUE)),
      directional_pct = mean(sign(estimate_h) == sign(estimate_l), na.rm = TRUE) * 100,
      inferential_pct = if (!anyNA(p_adj_h) && !anyNA(p_adj_l))
        mean(same_infer, na.rm = TRUE) * 100 else NA_real_,
      tost_pct        = mean(equivalent, na.rm = TRUE) * 100
    )
}

Distribution comparison metrics

compare_distributions() compares response distributions for a single condition and outcome using three metrics:

  • OVL (overlapping coefficient) — proportion of area where the two kernel density estimates overlap; 0 = no overlap, 1 = identical distributions; primary metric for RQ2
  • KS D-statistic — maximum absolute difference between the two empirical CDFs; 0 = identical
  • Variance ratio — LLM clone variance divided by human variance; 1 = equal spread
Code 
compute_ovl <- function(x, y, n_grid = 512) {
  lo <- min(c(x, y)); hi <- max(c(x, y))
  if (lo == hi) return(NA_real_)
  # Evaluate both KDEs on the same grid, then integrate the overlap area
  d_h <- density(x, from = lo, to = hi, n = n_grid)
  d_l <- density(y, from = lo, to = hi, n = n_grid)
  sum(pmin(d_h$y, d_l$y)) * (hi - lo) / n_grid
}

compare_distributions <- function(human_data, llm_data, outcome, condition_val) {
  x <- human_data |> filter(condition == condition_val) |> pull(all_of(outcome)) |> na.omit()
  y <- llm_data   |> filter(condition == condition_val) |> pull(all_of(outcome)) |> na.omit()
  tibble(
    condition      = as.character(condition_val),
    ovl            = compute_ovl(x, y),
    ks_d           = suppressWarnings(ks.test(x, y)$statistic),
    variance_ratio = var(y) / var(x)  # > 1: clones more variable; < 1: less variable
  )
}

Calibration regression

run_calibration() regresses human ATEs on LLM clone ATEs: ATE_human = α + β × ATE_clone. The intercept α captures systematic bias (positive α means clones underestimate the baseline, negative means overestimation). The slope β captures proportionality: β = 1 is perfect calibration, β > 1 means clone effects are too small relative to human effects, β < 1 means they are too large. We report a formal F-test of β = 1 alongside α, β, and R².

Code 
run_calibration <- function(h_result, l_result) {
  joined <- inner_join(
    h_result |> select(condition, estimate_h = estimate),
    l_result |> select(condition, estimate_l = estimate),
    by = "condition"
  )

  # ATE_h = α + β × ATE_l; perfect calibration: α = 0 and β = 1
  fit   <- lm(estimate_h ~ estimate_l, data = joined)
  coefs <- broom::tidy(fit, conf.int = TRUE)

  tibble(
    alpha     = coefs$estimate[1],
    alpha_lo  = coefs$conf.low[1],
    alpha_hi  = coefs$conf.high[1],
    beta      = coefs$estimate[2],
    beta_lo   = coefs$conf.low[2],
    beta_hi   = coefs$conf.high[2],
    r_squared = broom::glance(fit)$r.squared,
    # F-test H0: β = 1 (clone ATEs are proportionally calibrated to human ATEs)
    p_beta_1  = car::linearHypothesis(fit, "estimate_l = 1")$`Pr(>F)`[2]
  )
}

RQ4: Individual-level Demographic Calibration

## Research Questions

The main outcome of the megastudy is trust in climate scientists. However, there are several secondary and tertiary outcomes. For each of these outcomes we have the following research questions:

**RQ1 — ATE recovery.** How well do LLM clones predict the average treatment effects (ATEs) across 20 interventions?

**RQ2 — Distribution shape recovery.** How well do LLM clones mirror human response distributions (within experimental conditions)?

**RQ3 — Subgroup heterogeneity recovery.** How well do LLM clones recover differences in intervention effects across different demographic groups?

**RQ4 — Individual-level demographic calibration.** Are LLM clone responses calibrated to the demographic profile of each simulated participant? We examine (i) whether clones reproduce the correct baseline response level for each demographic group, (ii) whether demographic variables predict clone responses with the same magnitude as in humans (stereotyping test), and (iii) whether within-subgroup response distributions match between humans and clones.

**Secondary — Stacked-model discrepancy test.** As a complement to the TOST-based equivalence tests in RQ1, we pool human and clone data and estimate a condition × source interaction, providing a direct inferential test of whether each clone ATE significantly departs from its human counterpart.

## Comparison Functions

Two helper functions operationalize the comparison metrics. They are applied symmetrically to human and LLM clone model outputs throughout the document.

### Comparison metrics

`compare_estimates()` is used throughout the document — for ATEs (RQ1), moderator interaction estimates (RQ3), and secondary analyses. It takes two tidy model result tibbles and a `join_by` argument specifying the key columns, and computes six metrics:

- **Spearman ρ** — rank correlation of point estimates; primary metric for RQ1
- **Pearson r** — linear correlation of point estimates
- **RMSE** — root mean squared error; captures magnitude accuracy
- **Directional agreement** — % of estimates with the same sign
- **Inferential agreement** — % of estimates with the same BH-adjusted conclusion (significantly positive, significantly negative, or non-significant at α = .05)
- **TOST (%)** — % of individual estimate pairs that pass an equivalence test with preregistered bound Δ_k = 0.5 × |estimate_human_k|; equivalence is declared (two one-sided tests, α = .05) when the clone estimate falls within ±50% of the human estimate. Note: when human estimates are near zero, Δ_k shrinks accordingly, making equivalence harder to achieve.

```{r}
#| label: fn-compare-estimates

compare_estimates <- function(h_result, l_result,
                              join_by = "condition",
                              tost_delta_pct = 0.5) {

  # Standardize columns; SE derived from std.error or from 95% CI width if absent
  prep <- function(df, suffix) {
    df |>
      select(all_of(join_by), estimate) |>
      mutate(
        se    = if ("std.error" %in% names(df)) df$std.error
                else if (all(c("conf.low", "conf.high") %in% names(df)))
                  (df$conf.high - df$conf.low) / (2 * 1.96)
                else NA_real_,
        p_adj = if ("p.value_adjusted" %in% names(df)) df$p.value_adjusted
                else NA_real_
      ) |>
      rename_with(~ paste0(.x, "_", suffix), c(estimate, se, p_adj))
  }

  joined <- inner_join(prep(h_result, "h"), prep(l_result, "l"), by = join_by)

  # Three-level inferential category based on BH-adjusted p and direction
  infer_cat <- function(est, p_adj) {
    case_when(
      p_adj < 0.05 & est > 0 ~ "sig_positive",
      p_adj < 0.05 & est < 0 ~ "sig_negative",
      TRUE                   ~ "not_significant"
    )
  }

  # TOST: equivalence declared when |ATE_h - ATE_l| < delta = tost_delta_pct × |ATE_h|
  # p_tost is the maximum of the two one-sided p-values; equivalent if p_tost < 0.05
  joined |>
    mutate(
      delta      = tost_delta_pct * abs(estimate_h),
      diff       = estimate_h - estimate_l,
      se_diff    = sqrt(se_h^2 + se_l^2),
      p_lower    = pnorm((diff + delta) / se_diff, lower.tail = FALSE),
      p_upper    = pnorm((diff - delta) / se_diff),
      p_tost     = pmax(p_lower, p_upper),
      equivalent = p_tost < 0.05,
      same_infer = infer_cat(estimate_h, p_adj_h) == infer_cat(estimate_l, p_adj_l)
    ) |>
    summarise(
      spearman_rho    = cor(estimate_h, estimate_l, method = "spearman",
                            use = "pairwise.complete.obs"),
      pearson_r       = cor(estimate_h, estimate_l, use = "pairwise.complete.obs"),
      rmse            = sqrt(mean((estimate_h - estimate_l)^2, na.rm = TRUE)),
      directional_pct = mean(sign(estimate_h) == sign(estimate_l), na.rm = TRUE) * 100,
      inferential_pct = if (!anyNA(p_adj_h) && !anyNA(p_adj_l))
        mean(same_infer, na.rm = TRUE) * 100 else NA_real_,
      tost_pct        = mean(equivalent, na.rm = TRUE) * 100
    )
}
```

### Distribution comparison metrics

`compare_distributions()` compares response distributions for a single condition and outcome using three metrics:

- **OVL (overlapping coefficient)** — proportion of area where the two kernel density estimates overlap; 0 = no overlap, 1 = identical distributions; primary metric for RQ2
- **KS D-statistic** — maximum absolute difference between the two empirical CDFs; 0 = identical
- **Variance ratio** — LLM clone variance divided by human variance; 1 = equal spread

```{r}
#| label: fn-compare-distributions

compute_ovl <- function(x, y, n_grid = 512) {
  lo <- min(c(x, y)); hi <- max(c(x, y))
  if (lo == hi) return(NA_real_)
  # Evaluate both KDEs on the same grid, then integrate the overlap area
  d_h <- density(x, from = lo, to = hi, n = n_grid)
  d_l <- density(y, from = lo, to = hi, n = n_grid)
  sum(pmin(d_h$y, d_l$y)) * (hi - lo) / n_grid
}

compare_distributions <- function(human_data, llm_data, outcome, condition_val) {
  x <- human_data |> filter(condition == condition_val) |> pull(all_of(outcome)) |> na.omit()
  y <- llm_data   |> filter(condition == condition_val) |> pull(all_of(outcome)) |> na.omit()
  tibble(
    condition      = as.character(condition_val),
    ovl            = compute_ovl(x, y),
    ks_d           = suppressWarnings(ks.test(x, y)$statistic),
    variance_ratio = var(y) / var(x)  # > 1: clones more variable; < 1: less variable
  )
}
```

### Calibration regression

`run_calibration()` regresses human ATEs on LLM clone ATEs: `ATE_human = α + β × ATE_clone`. The intercept α captures systematic bias (positive α means clones underestimate the baseline, negative means overestimation). The slope β captures proportionality: β = 1 is perfect calibration, β > 1 means clone effects are too small relative to human effects, β < 1 means they are too large. We report a formal F-test of β = 1 alongside α, β, and R².

```{r}
#| label: fn-calibration

run_calibration <- function(h_result, l_result) {
  joined <- inner_join(
    h_result |> select(condition, estimate_h = estimate),
    l_result |> select(condition, estimate_l = estimate),
    by = "condition"
  )

  # ATE_h = α + β × ATE_l; perfect calibration: α = 0 and β = 1
  fit   <- lm(estimate_h ~ estimate_l, data = joined)
  coefs <- broom::tidy(fit, conf.int = TRUE)

  tibble(
    alpha     = coefs$estimate[1],
    alpha_lo  = coefs$conf.low[1],
    alpha_hi  = coefs$conf.high[1],
    beta      = coefs$estimate[2],
    beta_lo   = coefs$conf.low[2],
    beta_hi   = coefs$conf.high[2],
    r_squared = broom::glance(fit)$r.squared,
    # F-test H0: β = 1 (clone ATEs are proportionally calibrated to human ATEs)
    p_beta_1  = car::linearHypothesis(fit, "estimate_l = 1")$`Pr(>F)`[2]
  )
}
```

### RQ4c: Within-subgroup distribution shapes

We extend RQ2 by comparing response distributions within demographic subgroups in
the control condition. This tests whether distributional calibration holds within
groups, not just in the aggregate (a clone dataset could match the overall
distribution while having miscalibrated within-group spreads).

```{r}
#| label: rq4c-compute

control_val <- levels(human_data$condition)[1]

# For each moderator × level, compare human and clone distributions in control
subgroup_dist <- map_dfr(moderators_cat, function(mod) {
  groups <- levels(human_data[[mod]])
  map_dfr(groups, function(grp) {
    x <- human_data |>
      filter(condition == control_val, .data[[mod]] == grp) |>
      pull(trust_multidimensional) |> na.omit()
    y <- llm_data |>
      filter(condition == control_val, .data[[mod]] == grp) |>
      pull(trust_multidimensional) |> na.omit()
    # Require minimum n for stable KDE
    if (length(x) < 10 | length(y) < 10) return(NULL)
    tibble(moderator = mod, group = grp,
           ovl = compute_ovl(x, y),
           ks_d = suppressWarnings(ks.test(x, y)$statistic),
           variance_ratio = var(y) / var(x))
  })
})
```

```{r}

#| tbl-cap: "Within-subgroup distribution comparison in the control condition. OVL: overlapping coefficient (1 = identical). KS D: Kolmogorov–Smirnov statistic. Variance ratio: clone / human."

  arrange(moderator, group) |>
  gt(groupname_col = "moderator") |>
  cols_label(group = "Group", ovl = "OVL", ks_d = "KS D",
             variance_ratio = "Variance ratio") |>
  fmt_number(columns = c(ovl, ks_d, variance_ratio), decimals = 3) |>
  tab_style(style = cell_text(weight = "bold"),
```
